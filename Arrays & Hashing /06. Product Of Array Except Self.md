# 06. Product Of Array Except Self

## Problem

Given an integer array `nums`, return an array `answer` such that:

```text id="u2wly1"
answer[i]
=
product of all elements of nums
except nums[i]
```

Constraints:

* Must run in `O(n)` time.
* Cannot use division.

---

# Pattern

**Prefix Sum / Suffix Sum Pattern (Prefix Product + Suffix Product)**

Whenever the question says:

```text id="h8z59z"
For every index,
calculate answer using
elements on the left and right.
```

Think:

```text id="whhclw"
Prefix + Suffix
```

This pattern also appears in:

* Trapping Rain Water
* Maximum Product Subarray
* Prefix Sum Problems
* Equilibrium Index Problems

---

# Brute Force

For every index:

```text id="gnt3zc"
Multiply all elements except itself.
```

# Key Observation

For every index:

```text id="a8fcqv"
answer[i]
=
(product of left side)
*
(product of right side)
```

Example:

```text id="v5u5by"
nums = [1,2,3,4]

For i = 2:

Left Product
=
1 × 2
=
2

Right Product
=
4

Answer
=
2 × 4
=
8
```

# Approach 1 : Prefix + Suffix Arrays

## Prefix Array

```text id="8r7mh3"
nums = [1,2,3,4]

prefix = [1,1,2,6]
```

Meaning:

```text id="ngm6bn"
prefix[i]
=
product of all elements
before i
```

---

## Suffix Array

```text id="2btl9y"
suffix = [24,12,4,1]
```

Meaning:

```text id="nbklza"
suffix[i]
=
product of all elements
after i
```

---

## Answer

```text id="3nvg4l"
answer[i]
=
prefix[i]
*
suffix[i]
```

Result:

```text id="e72qea"
[24,12,8,6]
```

---

# Prefix + Suffix Code

```java id="v7st4n"
public int[] productExceptSelf(int[] nums) {

    int n = nums.length;

    int[] prefix = new int[n];
    int[] suffix = new int[n];
    int[] ans = new int[n];

    prefix[0] = 1;

    for(int i=1;i<n;i++)
    {
        prefix[i] =
                prefix[i-1] *
                nums[i-1];
    }

    suffix[n-1] = 1;

    for(int i=n-2;i>=0;i--)
    {
        suffix[i] =
                suffix[i+1] *
                nums[i+1];
    }

    for(int i=0;i<n;i++)
    {
        ans[i] =
                prefix[i] *
                suffix[i];
    }

    return ans;
}
```

# Optimal Approach (O(1) Extra Space)

## Key Observation

Do we really need:

```text id="6p5dhv"
prefix[]
suffix[]
```

No.

We can store prefix products directly in:

```text id="v72i0c"
answer[]
```

and keep only one variable:

```java id="ty0f7t"
rightProduct
```

for suffix products.

---

# Optimal Code

```java id="yudl9f"
public int[] productExceptSelf(int[] nums) {

    int n = nums.length;

    int[] ans = new int[n];

    ans[0] = 1;

    for(int i=1;i<n;i++)
    {
        ans[i] =
                ans[i-1] *
                nums[i-1];
    }

    int rightProduct = 1;

    for(int i=n-1;i>=0;i--)
    {
        ans[i] *= rightProduct;
        rightProduct *= nums[i];
    }

    return ans;
}
```

# Mistakes / Learnings

## Mistake 1

Initial Prefix:

```java id="rrv82n"
pre[0] = nums[0];
```

This stores:

```text id="a40u8j"
product including itself
```

which forces edge cases.

---

Better:

```java id="pmx2y1"
prefix[0] = 1;
```

Meaning:

```text id="yx90on"
No elements on the left.
```

---

## Mistake 2

Initial Suffix:

```java id="j2m5n1"
suffix[n-1] = nums[n-1];
```

Again includes itself.

Better:

```java id="3x8f3z"
suffix[n-1] = 1;
```

Meaning:

```text id="jsqgkl"
No elements on the right.
```

---

## Mistake 3

Your first solution:

```text id="4jmf61"
pre[] + suff[] + result[]
```

used:

```

extra space.

Interviewer follow-up:

```

Answer:
