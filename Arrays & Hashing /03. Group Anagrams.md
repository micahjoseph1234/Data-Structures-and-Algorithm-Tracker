# 03. Group Anagrams

## Problem

Given an array of strings `strs`, group the anagrams together.

Example:

```text
Input  : ["eat","tea","tan","ate","nat","bat"]
Output : [["eat","tea","ate"],["tan","nat"],["bat"]]
```

---

# Pattern

**HashMap + Signature Generation**

Whenever the question says:

```text
Group strings having same characters
```

Think:

```text
Generate a unique key (signature) for each string.
```

Then:

```text
signature -> list of strings
```

---

# Approach 1 : Sorting (Brute Force)

Sort every string and use the sorted string as the key.

Example:

```text
eat -> aet
tea -> aet
ate -> aet
```

HashMap:

```text
aet -> [eat, tea, ate]
ant -> [tan, nat]
abt -> [bat]
```

---

## Code Pattern

```java
char[] arr = s.toCharArray();
Arrays.sort(arr);
String key = new String(arr);
```

---

## Time Complexity

Sorting each string:

```text
O(k log k)
```

For `n` strings:

```text
O(n × k log k)
```

where

```text
k = average string length
```

---

## Space Complexity

```text
O(nk)
```

---

# Optimal Approach : Frequency Count Signature

Since strings contain lowercase letters:

```text
26 characters only
```

Generate a frequency array.

Example:

```text
eat

a -> 1
e -> 1
t -> 1
```

Key:

```text
#1#0#0#0#1....#1
```

Every anagram produces the same key.

---

# Algorithm

For every string:

### Step 1

Create frequency array.

```java
int[] freq = new int[26];
```

---

### Step 2

Count characters.

```java
freq[ch - 'a']++;
```

---

### Step 3

Build signature.

```java
StringBuilder sb = new StringBuilder();
```

```java
sb.append("#");
sb.append(freq[i]);
```

---

### Step 4

Store in HashMap.

```java
key -> list of anagrams
```

---

# Code

```java
public List<List<String>> groupAnagrams(String[] strs) {

    HashMap<String, List<String>> map =
            new HashMap<>();

    for (String s : strs) {

        int[] freq = new int[26];

        for (int i = 0; i < s.length(); i++) {
            freq[s.charAt(i) - 'a']++;
        }

        StringBuilder sb =
                new StringBuilder();

        for (int i = 0; i < 26; i++) {
            sb.append("#");
            sb.append(freq[i]);
        }

        String key = sb.toString();

        map.putIfAbsent(
                key,
                new ArrayList<>()
        );

        map.get(key).add(s);
    }

    return new ArrayList<>(map.values());
}
```

# Time Complexity

For each string:

```text
Count chars : O(k)
Build key   : O(26)
```

For `n` strings:

```text
O(n × k)
```

because 26 is constant.

---

# Space Complexity

```text
O(n × k)
```

for storing the grouped strings.

# Important Java Concepts

### Frequency Counting

```java
freq[ch - 'a']++;
```

---

### StringBuilder

```java
sb.append("#");
sb.append(freq[i]);
```

---

### Why '#'?

Without separator:

```text
111
```

Could mean:

```text
1,11
```

or

```text
11,1
```

Using:

```text
#1#11
```

removes ambiguity.

# Mistakes / Learnings

## Mistake 1

Initially forgot to add the string:

```java
map.get(key).add(s);
```

inside:

```java
if(!map.containsKey(key))
```

Result:

Lists remained empty.

---

## Mistake 2

Used:

```java
new ArrayList(map.values())
```

Prefer:

```java
new ArrayList<>(map.values())
```

Use generics.

---

## Mistake 3

Confusion between:

```java
strs.length
```

and

```java
list.size()
```

Remember:

```text
Array      -> length
String     -> length()
List       -> size()
```
