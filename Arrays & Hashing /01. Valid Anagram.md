# 01. Valid Anagram

## Problem

Given two strings `s` and `t`, return `true` if `t` is an anagram of `s`, and `false` otherwise.

---

## Pattern

* HashMap
* Frequency Counting

---

## Brute Force Approach

1. Sort both strings.
2. Compare the sorted strings.

### Time Complexity

```text
O(n log n)
```

### Space Complexity

```text
O(1)
```

---

## Optimal Approach (HashMap)

### Idea

* If lengths are different, return `false`.
* Store the frequency of each character of `s`.
* Traverse `t` and decrease the frequency.
* If frequency becomes `0`, remove the character from the map.
* At the end, if the map is empty, both strings are anagrams.

---

## Algorithm

1. Check if lengths are different.
2. Create a `HashMap<Character, Integer>`.
3. Count frequencies of characters in `s`.
4. Traverse `t`:

   * If character exists:

     * decrement frequency.
     * remove if frequency becomes `0`.
5. Return `map.size() == 0`.

---

## Code

```java
public boolean isAnagram(String s, String t) {

    if (s.length() != t.length()) {
        return false;
    }

    HashMap<Character, Integer> map = new HashMap<>();

    for (char c : s.toCharArray()) {
        map.put(c, map.getOrDefault(c, 0) + 1);
    }

    for (char c : t.toCharArray()) {
        if (!map.containsKey(c)) {
            return false;
        }

        int freq = map.get(c);

        if (freq == 1) {
            map.remove(c);
        } else {
            map.put(c, freq - 1);
        }
    }

    return map.isEmpty();
}
```

---

## Time Complexity

```text
O(n)
```

## Space Complexity

```text
O(1)
```

## Mistakes / Learnings

### Length Check

Instead of:

```java
if(n1 > n2 || n2 > n1)
```

prefer:

```java
if (n1 != n2)
```

### Frequency Pattern to Memorize

```java
map.put(ch, map.getOrDefault(ch, 0) + 1);
```

This pattern appears in:

* Valid Anagram
* Top K Frequent Elements
* Group Anagrams
* Longest Consecutive Sequence (frequency variant)

