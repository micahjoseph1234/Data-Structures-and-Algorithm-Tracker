# 02. Two Sum

## Problem

Given an array of integers `nums` and an integer `target`, return the indices of the two numbers such that they add up to the target.

---

# Pattern

**Array + HashMap (Complement Lookup)**

Whenever you hear:

```text
Find two elements whose sum equals X
```

Think:

```text
HashMap -> Value to Index Mapping
```

---

# Brute Force Approach

For every element:

* Check every other element.
* If their sum equals target, return indices.

## Time Complexity

```text
O(n²)
```

## Space Complexity

```text
O(1)
```

---

# Optimal Approach (HashMap)

## Key Observation

If:

```text
a + b = target
```

then:

```text
b = target - a
```

For every number:

1. Calculate complement.
2. Check if complement already exists.
3. If yes, return indices.
4. Otherwise, store current number and index.

---

# Algorithm

For each index `i`:

```text
complement = target - nums[i]
```

If complement exists in map:

```text
return [map.get(complement), i]
```

Else:

```text
map.put(nums[i], i)
```

---

# Code

```java
public int[] twoSum(int[] nums, int target) {

    HashMap<Integer, Integer> map = new HashMap<>();

    for (int i = 0; i < nums.length; i++) {

        int complement = target - nums[i];

        if (map.containsKey(complement)) {
            return new int[]{
                    map.get(complement),
                    i
            };
        }

        map.put(nums[i], i);
    }

    return new int[]{};
}
```

---

# Dry Run

```text
nums = [2,7,11,15]
target = 9
```

### i = 0

```text
num = 2
complement = 7
```

Map:

```text
2 -> 0
```

---

### i = 1

```text
num = 7
complement = 2
```

Map contains `2`.

Answer:

```text
[0,1]
```

---

# Time Complexity

```text
O(n)
```

Because each HashMap operation is O(1) on average.

---

# Space Complexity

```text
O(n)
```

In the worst case, all elements are stored.

# Mistakes / Learnings

## Mistake 1

First approach:

```java
Build entire HashMap
Then search for complement.
```

Problem:

```text
nums = [3,3]
target = 6
```

May use the same index twice.

---

## Correct Approach

```text
Check complement first.
Then insert current element.
```

Pattern:

```java
if(map.containsKey(complement))
{
    return ...
}

map.put(nums[i], i);
```

Memorize this.

# Key Takeaway

Whenever the question says:

```text
Find pair with sum = target
```

Immediately think:

```text
Complement + HashMap
```

This pattern appears in:

* Two Sum
* Two Sum II
* Four Sum
* Subarray Sum Equals K
* Count Pair Problems
